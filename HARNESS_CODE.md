# Coding Harness Specification

## 목적
본 프로젝트는 AI 생성 영상 탐지 모델의 베이스라인을 구현하기 위한 코드 자동화 및 구조 표준을 정의한다.  
모든 코드 작성은 일관된 구조, 재현 가능성, 검증 가능성을 보장해야 한다.

## 기본 원칙
- 모든 기능은 **테스트 가능해야 한다**
- 모든 코드 변경은 **로그로 남아야 한다**
- 결과는 **정해진 출력 형식(Output Spec)**을 반드시 따른다
- 임의 구현 금지 (TASK 기반 개발)

## 개발 흐름
1. TASK.md 확인
2. 계획 수립 → logs/plan.md 기록
3. 코드 작성
4. 테스트 실행
5. 결과 검증
6. logs 업데이트

## 금지 사항
- 테스트 없이 코드 작성 금지
- 출력 형식 무시 금지
- 하드코딩 금지
- 데이터 누수 가능성 있는 구현 금지

## 코드 스타일
- Python 기준 PEP8 준수
- 함수 단위로 역할 분리
- Docstring 필수

## 디렉토리 규칙
- src/: 실제 구현 코드
- tests/: 테스트 코드
- logs/: 실험 및 변경 기록
- docs/: 설계 문서

## 성공 기준
- 베이스라인 모델 학습 가능
- inference 정상 동작
- 재현 가능 (seed 고정)
