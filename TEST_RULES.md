# Test Rules

## 기본 원칙
- 모든 기능은 테스트 가능해야 한다
- 테스트 없이 merge 금지

## 테스트 종류

### Unit Test
- 함수 단위 테스트
- 입력/출력 검증

### Integration Test
- 데이터 → 모델 → 출력 흐름 확인

### Sanity Test
- 작은 데이터로 빠른 테스트

## 테스트 기준
- 에러 없이 실행
- 출력 형식 준수
- 성능 최소 기준 만족

## 실패 기준
- NaN 발생
- loss 폭증
- 출력 형식 불일치

## 추가 규칙
- 랜덤성 제거 (seed 고정)
- GPU/CPU 동일 동작 확인
